# Weekly review

# Overview
Provide the user with a detailed summary of all changes made to the codebase within the last 7-10 days

# Workflow
- Go through all commits and merges for the past 7~10 days (on the main repo branch)
- Don't read just the titles, read into each commit / merge 
- Create a synthesis of everything that was shipped based on the commits and most relevant changes
- Print 2~3 short paragraphs
- Also, highlight what you believe was tech debt, what seemed like bug fixes, and what looked like net new functionality in a short paragraph at the end

# Output Format
- Overview
- Main changes by category
- Notes and Suggestions
